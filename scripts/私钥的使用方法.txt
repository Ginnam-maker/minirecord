私钥使用方法（离线激活码签发）

一、文件位置
1) 私钥文件：scripts/secrets/license_private_key.pem
2) 签发脚本：scripts/generate-license-code.js
3) npm 命令：npm run license:gen

二、安全要求（务必遵守）
1) 私钥只能在本地离线环境使用，不要上传到任何仓库、网盘、聊天工具。
2) 私钥文件已在 .gitignore 中忽略：scripts/secrets/license_private_key.pem。
3) 建议再备份一份到加密U盘或密码管理器，避免丢失后无法继续签发激活码。
4) 一旦私钥泄露，应立即：
   - 生成新密钥对
   - 替换 utils/license.js 中公钥
   - 停止使用旧私钥签发的新码

三、生成激活码
1) 默认生成（365天、每月100次）：
   npm run license:gen

2) 指定有效天数（例如30天）：
   npm run license:gen -- --days 30

3) 指定月额度（例如每月100次）：
   npm run license:gen -- --limit 100

4) 指定绝对到期时间（毫秒时间戳）：
   npm run license:gen -- --expireAt 1777777777777

5) 永久码（无到期时间）：
   npm run license:gen -- --permanent

四、激活码格式
1) 输出格式：MR1.payload.signature
2) 在 App 总结页点击“激活付费版”，粘贴完整激活码即可。

五、参数说明
1) --days N
   从当前时间起 N 天后到期。
2) --limit N
   该激活码对应用户每月可提交周总结次数。
3) --expireAt TS
   使用绝对时间戳作为到期时间（毫秒）。
4) --permanent
   生成永久有效码（expireAt = null）。
5) --key PATH
   使用指定私钥文件路径（默认 scripts/secrets/license_private_key.pem）。

六、常见问题
1) 提示“私钥文件不存在”
   检查 scripts/secrets/license_private_key.pem 是否存在，或使用 --key 指定路径。

2) 提示“激活码签名无效”
   检查 utils/license.js 中公钥是否与该私钥成对。

3) 提示“激活码已过期”
   重新生成有效期更长的激活码（--days 或 --expireAt）。

七、建议流程（发码）
1) 本地离线执行签发命令
2) 复制输出激活码
3) 发给用户激活
4) 不要把私钥文件发给任何人
