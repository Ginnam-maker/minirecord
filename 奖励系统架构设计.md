# miniRecord 奖励系统架构设计

## 1. 文档目标

为 miniRecord 项目增加「奖励系统」能力，允许用户：

- 通过自然语言定义积分规则（如"读书五分钟加1分"）
- 系统自动解析生成结构化规则
- 每日保存记录时自动匹配规则、触发积分变动
- 查看积分历史和当前积分余额

本设计不改变现有记录和周总结能力，仅在保存记录流程中增加积分结算钩子。

---

## 2. 现状与约束

基于现有代码，奖励系统设计需遵循以下约束：

| 约束项 | 说明 |
|--------|------|
| 技术栈 | Vue 3 + uni-app + Pinia |
| 数据持久化 | 本地存储（`utils/storage.js`） |
| 现有 AI 接入 | DeepSeek API（`utils/deepseek.js`） |
| 记录入口 | `pages/record/list.vue` |
| 保存流程 | `recordStore.saveRecord(date, content)` |
| 配置规范 | `config.example.js` + 本地 `config.js` |
| 日志规范 | `utils/logger.js` |

---

## 3. 核心概念

### 3.1 术语定义

| 术语 | 定义 |
|------|------|
| **规则 (Rule)** | 用户定义的积分触发条件，包含条件表达式和积分变动值 |
| **条件 (Condition)** | 规则的触发判断逻辑，如"读书时间 > 5分钟" |
| **积分 (Points)** | 用户累积的奖励数值，可增可减 |
| **结算 (Settlement)** | 保存记录时匹配规则并计算积分变动的过程 |
| **NLU 解析** | 将自然语言转换为结构化规则的过程 |

### 3.2 规则类型

| 类型 | 描述 | 示例 |
|------|------|------|
| **时长类** | 基于活动时长触发 | "读书超过30分钟+2分" |
| **次数类** | 基于活动出现次数触发 | "运动了就+1分" |
| **关键词类** | 基于记录包含特定关键词触发 | "记录中提到'早起'+1分" |
| **否定类** | 基于未完成某事触发扣分 | "没运动-1分" |

```typescript
interface RewardRule {
  id: string                    // 唯一标识，如 'rule_1712345678901'

    unit?: 'minute' | 'hour' | 'count'
  }

}
```

## 6. 模块设计

### 6.1 规则解析器 (RuleParser)

**职责**：将用户自然语言输入转换为结构化规则

**位置**：`utils/rewardParser.js`

```javascript
/**
 * 解析自然语言规则
 * @param {string} text 用户输入，如 "读书五分钟加1分"
 * @returns {Promise<ParsedRule>} 结构化规则
 */
export async function parseNaturalRule(text) {
  // 1. 先尝试本地正则匹配（快速、离线）
  const localResult = tryLocalParse(text)
  if (localResult.confidence > 0.8) {
    return localResult.rule
  }
  
  // 2. 调用 LLM 进行智能解析（兜底）
  return await llmParse(text)
}
```

#### 6.1.1 本地解析策略

使用正则表达式匹配常见模式：

```javascript
const PATTERNS = [
  // 时长类：读书/运动 + 时间 + 加/减分
  {
    regex: /(?<activity>\S+?)(?<value>\d+|[一二三四五六七八九十]+)(分钟|小时|min|h)(?:以上)?(?<action>加|减|\+|-)(?<points>\d+|[一二三四五六七八九十]+)(分|积分)?/,
    type: 'duration'
  },
  
  // 次数类：做了某事 + 加分
  {
    regex: /(?<activity>\S+?)了?就?(?<action>加|减|\+|-)(?<points>\d+|[一二三四五六七八九十]+)(分|积分)?/,
    type: 'count'
  },
  
  // 关键词类：提到/包含 + 关键词
  {
    regex: /(?:提到|包含|有)['"]?(?<activity>\S+?)['"]?(?<action>加|减|\+|-)(?<points>\d+|[一二三四五六七八九十]+)(分|积分)?/,
    type: 'keyword'
  },
  
  // 否定类：没/未 + 某事 + 扣分
  {
    regex: /(?:没|未|没有)(?<activity>\S+?)(?<action>扣|减|-)(?<points>\d+|[一二三四五六七八九十]+)(分|积分)?/,
    type: 'absence'
  }
]
```

### 6.2 规则引擎 (RuleEngine)

**职责**：根据记录内容匹配规则、计算积分

**位置**：`utils/rewardEngine.js`

```javascript
/**
 * 匹配记录与规则
 * @param {string} recordContent 记录内容
 * @param {RewardRule[]} rules 规则列表
 * @returns {MatchResult[]} 匹配结果
 */
export function matchRules(recordContent, rules) {
  const results = []
  
  for (const rule of rules) {
    if (!rule.enabled) continue
    
    const match = evaluateCondition(recordContent, rule.condition)
    if (match.matched) {
      results.push({
        ruleId: rule.id,
        ruleName: rule.name,
        points: rule.points,
        reason: match.reason
      })
    }
  }
  
  return results
}
```

#### 6.2.1 条件评估逻辑

```javascript
function evaluateCondition(content, condition) {
  const { type, activity, operator, value, unit } = condition
  
  switch (type) {
    case 'duration':
      return evaluateDuration(content, activity, operator, value, unit)
    
    case 'count':
      return evaluateCount(content, activity)
    
    case 'keyword':
      return evaluateKeyword(content, activity)
    
    case 'absence':
      return evaluateAbsence(content, activity)
    
    default:
      return { matched: false, reason: '未知条件类型' }
  }
}
```

#### 6.2.2 时长提取算法

从记录中提取活动时长：

```javascript
const DURATION_PATTERNS = [
  // "读书30分钟" / "读书30min"
  /(?<activity>\S+?)(?<value>\d+)(分钟|min)/g,
  
  // "读书1小时" / "读书1h"
  /(?<activity>\S+?)(?<value>\d+)(小时|hour|h)/g,
  
  // "读书了半小时"
  /(?<activity>\S+?)了?半(小时|个钟)/g,
  
  // "9点到10点读书" (计算时间差)
  /(?<start>\d{1,2})[点:：](?<startMin>\d{0,2})?到(?<end>\d{1,2})[点:：](?<endMin>\d{0,2})?(?<activity>\S+)/g
]

function extractDuration(content, activity) {
  // 1. 尝试直接匹配时长
  for (const pattern of DURATION_PATTERNS) {
    const matches = content.matchAll(pattern)
    for (const match of matches) {
      if (match.groups.activity?.includes(activity)) {
        return parseDurationValue(match)
      }
    }
  }
  
  // 2. 活动存在但无时长，默认返回 null
  if (content.includes(activity)) {
    return null // 表示活动存在但时长不明
  }
  
  return 0 // 活动不存在
}
```

### 6.3 积分状态管理 (PointsStore)

**职责**：管理积分状态、持久化

**位置**：`stores/rewardStore.js`

```javascript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useRewardStore = defineStore('reward', () => {
  // 状态
  const rules = ref([])
  const points = ref(0)
  const history = ref([])
  
  // 计算属性
  const enabledRules = computed(() => 
    rules.value.filter(r => r.enabled)
  )
  
  const todayPoints = computed(() => {
    const today = formatDate(new Date())
    return history.value
      .filter(h => h.date === today)
      .reduce((sum, h) => sum + h.points, 0)
  })
  
  // 方法
  function init() { /* 加载本地数据 */ }
  function addRule(rule) { /* 添加规则 */ }
  function updateRule(id, updates) { /* 更新规则 */ }
  function deleteRule(id) { /* 删除规则 */ }
  function toggleRule(id) { /* 切换启用状态 */ }
  function settlePoints(date, content) { /* 结算积分 */ }
  
  return {
    rules, points, history,
    enabledRules, todayPoints,
    init, addRule, updateRule, deleteRule, toggleRule, settlePoints
  }
})
```

## 7. 流程设计

### 7.1 规则创建流程

```
用户输入自然语言 ──▶ RuleParser 解析 ──▶ 展示预览 ──▶ 用户确认 ──▶ 保存规则
       │                    │                │
       │                    ▼                │
       │              本地解析失败?          │
       │                    │                │
       │                    ▼                │
       │              调用 LLM 解析          │
       │                    │                │
       │                    ▼                │
       └──────────────▶ 解析失败 ◀──────────┘
                            │
                            ▼
                     提示用户重新输入
```

### 7.2 积分结算流程

```
保存记录 (recordStore.saveRecord)
       │
       ▼
  获取启用的规则列表
       │
       ▼
  遍历规则匹配记录内容
       │
       ├──▶ 规则1: 匹配成功 ──▶ +1分
       ├──▶ 规则2: 匹配失败 ──▶ 跳过
       └──▶ 规则3: 匹配成功 ──▶ -1分 (扣分规则)
       │
       ▼
  计算积分变动总和
       │
       ▼
  更新积分余额 & 记录历史
       │
       ▼
  返回结算结果 (用于 UI 展示)
```

### 7.3 与现有保存流程集成

修改 `recordStore.saveRecord`：

```javascript
// stores/recordStore.js
import { useRewardStore } from './rewardStore'

function saveRecord(date, content) {
  const success = saveRecordToStorage(date, content)
  if (success) {
    loadRecords()
    
    // 🆕 积分结算
    const rewardStore = useRewardStore()
    const settlement = rewardStore.settlePoints(date, content)
    
    return {
      success: true,
      settlement // { totalPoints, changes: [...] }
    }
  }
  return { success: false }
}
```

## 10. 错误处理

### 10.1 规则解析错误

| 错误场景 | 处理方式 |
|----------|----------|
| 本地解析失败 | 自动尝试 LLM 解析 |
| LLM 解析超时 | 提示"解析超时，请简化描述或稍后重试" |
| LLM 返回格式错误 | 提示"无法理解，请尝试其他表达方式" |
| 无效规则 | 提示具体问题，如"缺少积分数值" |

### 10.2 积分结算错误

| 错误场景 | 处理方式 |
|----------|----------|
| 规则数据损坏 | 跳过该规则，记录日志 |
| 存储失败 | 重试一次，仍失败则提示用户 |
| 历史记录过多 | 自动清理 6 个月前的记录 |

### 10.3 降级策略

- 奖励系统任何错误不影响记录保存主流程
- LLM 不可用时仅支持本地解析
- 本地存储满时自动清理旧历史

## 11. 性能考量

### 11.1 规则匹配优化

- 规则数量限制：默认最多 20 条
- 快速过滤：先检查 `activity` 关键词是否存在
- 延迟加载：规则列表按需加载

### 11.2 存储优化

- 历史记录分页加载
- 定期清理过期数据（>6个月）
- 积分余额与历史分开存储

### 11.3 UI 优化

- 结算弹窗可关闭
- 积分变动使用轻提示而非弹窗（可选）
- 规则列表虚拟滚动（规则多时）

## 16. 总结

本设计通过「自然语言解析 + 规则引擎 + 积分系统」三层架构，实现用户友好的奖励机制。核心特点：

1. **易用**：自然语言定义规则，无需学习复杂语法
2. **智能**：本地解析 + LLM 兜底，兼顾速度和准确性
3. **无侵入**：不修改现有记录数据结构，仅在保存流程增加钩子
4. **可扩展**：规则类型、积分玩法均可灵活扩展

建议按 M1 → M2 → M3 分阶段实施，先跑通核心流程，再逐步丰富功能。
